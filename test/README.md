# ProfSync Test Stack

Complete Docker-based test environment with Sonarr and Radarr for testing the ProfSync wizard.

## Quick Start

```bash
cd test/
./init-stack.sh
```

This will:
1. Start Sonarr and Radarr containers
2. Wait for both to be ready
3. Extract API keys from their config files
4. Configure root folders
5. Create `.env` with connection details
6. Import pre-populated test libraries (20 series, 20 movies)

## Running the Wizard

After initialization, test the wizard with:

```bash
cd ..
python wizard.py --teststack
```

In `--teststack` mode:
- Sonarr and Radarr connection details are auto-loaded from `test/.env`
- No need to manually enter URLs and API keys
- Perfect for testing the configuration flow

Or manually:

```bash
python wizard.py
# When prompted:
# Sonarr URL: http://localhost:8989
# Sonarr API Key: [from test/.env]
# Radarr URL: http://localhost:7878
# Radarr API Key: [from test/.env]
```

## Cleanup

```bash
cd test/
./cleanup-stack.sh
```

This will stop containers and optionally remove data volumes.

## Authentication

The test stack is configured with **authentication disabled** for simplicity:
- Web UIs are accessible without login at http://localhost:8989 and http://localhost:7878
- API calls work without additional authentication (only require the API key header)
- This is safe for local testing — do not use this config in production

If you need authentication for testing purposes, you can manually enable it by:
1. Accessing the web UI
2. Going to Settings → Security
3. Configuring username/password and bypass rules

## Files

- `docker-compose.yml` — Sonarr + Radarr container definitions
- `init-stack.sh` — Bootstrap and configure services
- `cleanup-stack.sh` — Stop and remove containers
- `.env` — Generated by init-stack.sh (API keys, URLs)
- `sonarr-config/`, `radarr-config/` — Service configuration volumes
- `sonarr-tv/`, `sonarr-downloads/` — Sonarr media/staging dirs
- `radarr-movies/`, `radarr-downloads/` — Radarr media/staging dirs

## Troubleshooting

**Services won't start:**
```bash
docker-compose logs sonarr
docker-compose logs radarr
```

**API key extraction fails:**
- Wait longer for services to initialize (check logs)
- Manually extract from config files:
  - `sonarr-config/config.xml` (look for `<ApiKey>`)
  - `radarr-config/config.xml` (look for `<ApiKey>`)

**Port conflicts:**
- Change ports in `docker-compose.yml` if 8989 or 7878 are in use
- Update corresponding URLs in `.env` after initialization
